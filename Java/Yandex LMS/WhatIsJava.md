# Знакомство с Java

## 1. Что такое Java?
- Язык программирования, созданный в **1995 году**.
- Используется в **финтехе, логистике, Android-разработке, бэкенде** и многих других сферах.
- Примеры задач: банковские сервисы, телеграм-боты, мобильные приложения, обработка видео.

---

## 2. Почему стоит учить Java?

| Особенность | Что даёт |
|-------------|----------|
| **Надёжность и безопасность** | Подходит для критически важных систем (банки, госструктуры) |
| **Производительность** | Выдерживает сотни тысяч запросов, позволяет тонко настраивать приложение |
| **Большая экосистема** | Огромное количество библиотек и готовых решений (базы данных, авторизация, веб-сервисы) |
| **Кросс-платформенность** | «Write once, run everywhere» - программа работает на любом устройстве без переписывания |

---

## 3. Java ≠ JavaScript

| Java | JavaScript |
|------|------------|
| **Бэкенд** (серверная часть) | **Фронтенд** (веб-интерфейсы, кнопки, анимации) |
| Статическая типизация | Динамическая типизация (можно смешивать типы) |
| Пример: `"5" – 3` → ошибка компиляции | Пример: `"5" – 3` → `2`, но `"5" + 3` → `"53"` |

> **Важно:** Java не позволит вычитать число из строки - это ошибка на этапе написания кода!

## 4. Ключевые особенности Java

### 4.1. Статическая типизация
- Каждая переменная **обязана** иметь заранее объявленный тип (целое число, строка, дробь и т.д.).
- Нельзя в одну переменную положить сначала число, а потом строку - компилятор это запретит.
- **Плюс:** многие ошибки отлавливаются ещё до запуска программы.

**Пример (сравнение):**
```python
# Python (динамическая)
x = 5
x = "текст"   # можно
```
```java
// Java (статическая)
int x = 5;
x = "текст";  // ОШИБКА! тип не совпадает
```

### 4.2. Объектно-ориентированность
- Всё в Java - объекты. Программа строится как взаимодействие объектов.
- Объекты имеют свойства и поведение.
- Пример: объект «Студент» содержит объекты «Курсы», те в свою очередь - «Задачи». Студент может «выполнить» задачу, меняя её статус.

### 4.3. Кроссплатформенность и этапы работы
Процесс превращения кода в работающую программу:

1. **Исходный код** - текст, написанный разработчиком на Java.
2. **Компилятор** (`javac`) переводит исходник в **байт-код** - специальный промежуточный набор инструкций.
3. **Java Virtual Machine (JVM)** - виртуальная машина, которая:
   - читает байт-код,
   - переводит его в **машинный код** для конкретного процессора,
   - запускает программу.

Благодаря JVM одна и та же программа (байт-код) работает на Windows, Linux, macOS - везде, где есть JVM.
