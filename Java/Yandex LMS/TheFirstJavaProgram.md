# Первая программа на Java

## 1. Структура первой программы

```java
public class Main {
    public static void main(String[] args) {
        // команды (инструкции)
    }
}
```

### Элементы объявления класса
- `public` - модификатор доступа, означает, что класс доступен из любой части программы.
- `class` - ключевое слово для объявления класса.
- `Main` - имя класса (может быть любым, но принято начинать с заглавной буквы, стиль `UpperCamelCase`).
- `{ ... }` - тело класса, в котором пишутся методы и поля.

### Элементы объявления метода `main`
- `public` - модификатор доступа (метод виден везде).
- `static` - метод принадлежит классу, а не объекту (вызывается без создания экземпляра).
- `void` - метод не возвращает никакого значения.
- `main` - **имя метода**, именно его Java ищет как точку входа при запуске программы.
- `(String[] args)` - параметр - массив строк, переданных из командной строки (пока не используется).
- `{ ... }` - тело метода, содержащее выполняемые команды.

> **Важно:** Имя класса (`Main`) и имя метода (`main`) никак не связаны. Метод входа всегда называется `main`, независимо от имени класса.

## 2. Вывод в консоль

Команда для вывода текста с переводом строки:
```java
System.out.println("Текст");
```

Пример программы, выводящей две строки:
```java
public class Main {
    public static void main(String[] args) {
        System.out.println("Hello world!");
        System.out.println("Учим Java шаг за шагом");
    }
}
```

---

## 3. Обязательные правила синтаксиса

### Точка с запятой `;`
- Ставится **в конце каждой команды** (инструкции).
- **Не ставится** после объявления класса и метода (после `{` и `}`).
- Команда может занимать несколько строк, но точка с запятой ставится один раз в конце всей команды.

```java
System.out.println("Hello");   // правильно
int x = 5;                     // правильно
// public class Main { ... }   // после } точки с запятой нет
```

### Фигурные скобки `{ }`
- Обозначают **блок кода** - группу инструкций, принадлежащих классу, методу или управляющей конструкции.
- Открывающая `{` и закрывающая `}` скобки обязательны для тела класса и метода.

## 4. Общепринятые правила форматирования (для читаемости)

- **Отступы** (обычно 4 пробела) показывают вложенность.
```java
public class Main {
    public static void main(String[] args) {
        System.out.println("Hello");   // отступ внутри метода
    }
}
```
- **Не пишите всё в одну строку** - код должен быть структурирован.
- Компилятор игнорирует лишние пробелы и переносы строк, но для людей они важны.

## 5. Комментарии

- Однострочный: `// текст комментария`
- Многострочный: `/* текст комментария */`
- Комментарии не влияют на работу программы, служат для пояснений.

```java
public static void main(String[] args) {
    // Это однострочный комментарий
    /*
       Это многострочный
       комментарий
    */
    System.out.println("Hello");
}
```

## 6. Переменные

**Переменная** - именованная ячейка памяти для хранения значения.

### Объявление и инициализация

```java
int step;          // объявление (тип int, имя step)
step = 3;          // присваивание значения
int step2 = 10;    // объявление + инициализация (сразу со значением)
```

### Изменение значения
```java
int step = 3;
System.out.println(step);   // 3
step = 10;                  // новое значение
System.out.println(step);   // 10
step = step + 2;            // арифметика с переменной
System.out.println(step);   // 12
```

### Использование переменной в выводе
```java
String text = "Hello world!";
System.out.println(text);   // выведет Hello world!
```

### Правила именования переменных
- **Чувствительны к регистру**: `step` и `Step` - разные имена.
- Могут начинаться с **буквы**, `$` или `_`. Для своих программ используйте **букву**.
- Далее могут быть буквы, цифры, `$`, `_`.
- **Нельзя** начинать с цифры и использовать дефис (`-`).
- Принят стиль **lowerCamelCase**: `stepByStep`, `myVariableName`.
- Для классов используется **UpperCamelCase**: `Main`, `MyFirstClass`.

**Примеры корректных имён:** `step3`, `level0`, `$temp`, `_private`.
**Некорректные:** `3step` (начинается с цифры), `step-3` (дефис).

## 7. Тип данных `int` и `String`
- `int` - целые числа (integer).
- `String` - строки (последовательность символов). Записываются в двойных кавычках: `"текст"`.

> В Java каждая переменная обязана иметь явно объявленный тип (статическая типизация). Тип нельзя изменить после объявления.
