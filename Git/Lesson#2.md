# Основы работы с Git

## Компоненты Git

### Рабочий каталог (Working Directory)
* **Единичная проверка** одной версии проекта
* Файлы извлекаются из локальной базы данных
* Место для редактирования файлов проекта
* Текущая рабочая версия проекта

### Промежуточная область (Index/Stage)
* **Техническое название**: Index
* Находится в каталоге .git
* Хранит информацию о следующем коммите
* Содержит файлы, подготовленные к фиксации

### Область хранилища (Repository)
* **Локальная база данных** проекта
* Хранится в папке .git
* Содержит метаданные и объекты проекта
* Основная часть Git

## Состояния файлов в Git

### Modified (Изменённый)
* Файл был модифицирован
* Изменения не добавлены в индекс
* Находится в рабочем каталоге

### Staged (Промежуточный)
* Изменения добавлены в индекс
* Файл готов к фиксации
* Находится в промежуточной области

### Committed (Зафиксированный)
* Изменения сохранены в репозиторий
* Являются частью истории проекта

## Рабочий процесс

1. **Изменение файлов**
   * Модификация файлов в рабочем каталоге
   * Выбор файлов для включения в следующий коммит

2. **Подготовка к коммиту**
   * Добавление изменений в индекс
   * Формирование содержимого коммита

3. **Фиксация изменений**
   * Сохранение моментального снимка
   * Запись изменений в локальную базу данных

## Основные термины

* **Репозиторий** (Repository) - хранилище всех версий проекта
* **Индекс** (Index) - область подготовки изменений
* **Рабочая копия** (Working Copy) - текущая версия проекта
* **Коммит** (Commit) - фиксация изменений
* **Промежуточная область** (Stage) - зона подготовки файлов
* **Локальная база данных** - хранилище метаданных в папке .git
