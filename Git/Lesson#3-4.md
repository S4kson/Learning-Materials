## Конфигурация Git

### Основные моменты

**Git конфигурация** включает три уровня настроек:

* **Уровень проекта** - локальный уровень конфигурации
* **Уровень пользователя** - настройки для конкретного пользователя системы
* **Системный уровень** - глобальные настройки для всей системы

### Расположение конфигурационных файлов

| Уровень | Расположение |
|---------|--------------|
| **Проект** | В папке `.git` проекта: `./.git/config` |
| **Пользователь** | В домашней директории: `~/.gitconfig` или `C:\Users\<ИмяПользователя>\.gitconfig` |
| **Системный** | В директории установки Git: `C:\Program Files\Git\etc\gitconfig` |

### Важные команды

* **Проверка версии Git**:
```bash
git --version
```

* **Просмотр конфигурации**:
```bash
git config --list --show-origin
```

### Приоритет конфигурации

Порядок проверки настроек:
1. Сначала проверяется **уровень проекта**
2. Затем **уровень пользователя**
3. В конце **системный уровень**

### Зачем нужна конфигурация

**Имя пользователя и email** в Git необходимы для:
* Идентификации автора изменений
* Привязки коммитов к конкретному разработчику
* Формирования истории изменений проекта

### Практическое применение

Перед началом работы с Git необходимо настроить:
* Имя пользователя
* Адрес электронной почты
* Другие параметры конфигурации

Эти настройки помогут правильно отслеживать изменения в проекте и идентифицировать автора каждого коммита.

### Примечания

* Настройки на более высоком уровне (проект) переопределяют настройки с более низкого уровня
* Рекомендуется настраивать имя пользователя и email на **пользовательском уровне** для единообразия во всех проектах
