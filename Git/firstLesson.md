# Введение в Git и системы контроля версий

## Что такое Git

**Git** — это распределённая система контроля версий (VCS), разработанная Линусом Торвальдсом для работы над ядром Linux.

## Основные понятия

**Система контроля версий (VCS)** — программное обеспечение для:
* Записи изменений в файлах
* Отслеживания истории правок
* Восстановления предыдущих версий
* Работы с разными типами файлов (код, изображения, документы)

## Типы систем контроля версий

### 1. Локальная VCS

**Принцип работы:**
* Копирование файлов в отдельную папку
* Добавление временных меток к копиям

**Характеристики:**
* Простая реализация
* Индивидуальная работа
* Хранение данных на локальном компьютере

**Недостатки:**
* Риск потери данных
* Сложности с управлением версиями
* Отсутствие совместной работы

### 2. Централизованная VCS

**Принцип работы:**
* Единый сервер для хранения всех версий
* Клиенты получают файлы из центрального хранилища

**Примеры:**
* CBS
* Subversion
* Perforce

**Особенности:**
* Централизованное управление доступом
* Мгновенные обновления
* Контроль действий пользователей

**Недостатки:**
* Зависимость от интернета
* Риск потери данных при сбое сервера
* Остановка работы при недоступности сервера

### 3. Распределённая VCS

**Принцип работы:**
* Полная копия репозитория на каждом клиенте
* Локальное хранение истории изменений

**Преимущества:**
* Автономная работа
* Возможность синхронизации
* Каждый клиент - резервная копия
* Надёжное хранение данных

**Ключевые особенности:**
* Независимость от центрального сервера
* Сохранение полной истории на каждом клиенте
* Гибкость в работе

## Применение Git

**Git используется для:**
* Отслеживания изменений в коде
* Совместной разработки
* Управления проектами
* Работы с любыми типами файлов
* Создания резервных копий
