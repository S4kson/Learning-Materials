# Добавление и фиксация изменений в Git

## Области хранения изменений

* **Рабочая директория (Working Directory)** - файлы, которые редактируются в текущий момент.
* **Область подготовки (Staging Area / Index)** - промежуточное хранилище, куда помещаются изменения перед фиксацией. Позволяет отбирать, какие правки войдут в следующий коммит.
* **Репозиторий (Repository)** - место, где Git хранит зафиксированные снапшоты (коммиты). После коммита изменения становятся частью истории проекта.

## Команда добавления изменений (`git add`)

### Назначение
Переносит изменения из рабочей директории в область подготовки.

### Синтаксис и примеры
* `git add <имя_файла>` - добавить конкретный файл.  
  Пример: `git add index.html`
* `git add .` - добавить все изменённые и новые файлы в текущей папке.
* `git add -A` - добавить все изменения во всём проекте.

### Важные моменты
* В staging area фиксируется состояние файла **на момент выполнения команды**.  
* Если после `git add` файл снова изменить, новые правки не попадут в staging до повторного выполнения команды.

## Команда проверки состояния (`git status`)

### Назначение
Показывает текущее состояние рабочей директории и области подготовки.

### Секции вывода
* `Changes to be committed` - файлы, уже находящиеся в staging и готовые к коммиту.
* `Changes not staged for commit` - изменённые файлы, которые ещё не добавлены в staging.
* `Untracked files` - новые файлы, которые Git ещё не отслеживает.

## Команда фиксации изменений (`git commit`)

### Назначение
Сохраняет содержимое staging area в репозиторий как новый коммит.

### Синтаксис
`git commit -m "Краткое описание изменений"`

### Важно
* В коммит попадают **только те изменения, которые были добавлены в staging** на момент выполнения.  
* Всё, что осталось в рабочей директории без `git add`, не включается.

## Типовой цикл работы

1. Внести изменения в файлы в рабочей директории.
2. Проверить статус (`git status`) - увидеть, что изменилось.
3. Добавить нужные изменения в staging (`git add`).
4. Снова проверить статус - убедиться, что подготовлены нужные файлы.
5. Зафиксировать коммит (`git commit -m "..."`).
6. Изменения сохранены в репозитории.


## Основные команды (шпаргалка)

* `git add <файл>` - добавить файл в staging.
* `git add .` - добавить все изменения в текущей директории.
* `git status` - показать состояние рабочей директории и staging.
* `git commit -m "..."` - создать коммит из staging.
* `git log` - просмотреть историю коммитов.
